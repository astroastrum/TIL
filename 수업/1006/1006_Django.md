## Django 

* 장고는 웹 프레임워크(틀)
  1. URL로 요청을 받고
  2. 처리하고
  3. 응답을 한다



* Django ORM
  * 데이터베이스에서 값을 가지고 오고
  * 데이터베이스에 저장한다



* Django의 목표
  * 서비스 기능
    * 게시판 기능
      * 생성
        * 사용자에게 HTML Form을 제공
        * DB에 저장
      * 조회
        * 글을 클릭하면 DB에 값 조회
      * 삭제
        * 삭제 버튼을 클릭하면 삭제
      * 수정
        * HTML Form과 기존값을 제공
        * DB 저장 (기존 DB 수정 과정)



* HTTP
  * Path/메서드/헤더/프로토콜



* ModelForm
  * 사용자로부터 데이터를 받는데 DB 필드가 변할 때 HTML Form도 맞춰서 변화
  * Input 값을 DB로 저장하는 과정에는 반드시 유효성 검사가 필요
  * *DB 필드를 HTML Form으로 변화시키고 그 HTML Form에서 받는 데이터를 유효성 검사*
